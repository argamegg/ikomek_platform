# Web Request Flow QA

Use this checklist when request creation or request details change in the web app.

## Create Request

- Start from `/requests/new`.
- Choose a category.
- Enter an address, place type, problem type, reason, and description.
- Pick a map location if the flow requests coordinates.
- Submit and confirm the request appears in `/requests`.

## Request Details

- Open the newly created request.
- Confirm status, address, category, description, and timestamps are readable.
- Open the chat route and send a message if the backend is running.

## Operator Review

- Log in as operator.
- Open `/operator`.
- Confirm the new request is visible.
- Update status and confirm the citizen view reflects the change.
