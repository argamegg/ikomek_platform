# Demo Account Handling

Demo accounts are intended for local testing and presentations.

## Accounts

| Role | Email | Password |
| --- | --- | --- |
| Citizen | `demo@ikomek.kz` | `demo123` |
| Operator | `operator@ikomek.kz` | `operator123` |
| Admin | `admin@ikomek.kz` | `admin123` |

## Usage Guidelines

- Use demo accounts only in local or demo environments.
- Do not reuse demo passwords in production.
- Reset seed data when demos need a predictable starting state.
- Keep role expectations aligned with the QA role matrix.

## Demo Preparation

- Seed data before the demo.
- Confirm all three accounts can log in.
- Create one fresh citizen request.
- Confirm operator and admin accounts can see it.
